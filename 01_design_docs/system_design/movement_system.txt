### 2. 移動邏輯：進度指針系統

我們用一個變數 `current_floor` 來記錄玩家走到了第幾步。

```python
import random

def start_adventure():
    current_floor = 0
    hp = 100
    
    print("--- 冒險開始 ---")
    
    while current_floor < len(dungeons):
        location, level, desc = dungeons[current_floor]
        print(f"\n【目前位置：{location}】(進度: {current_floor + 1}/{len(dungeons)})")
        print(f"描述：{desc}")
        
        choice = input("你的行動？ (1. 繼續前進 / 2. 撤退回城): ")
        
        if choice == "1":
            # 隨機事件判定
            event_roll = random.random()
            if event_roll < 0.3: # 30% 機率遇敵
                print(">> 遭遇怪物！進入戰鬥...")
                # 這裡可以接戰鬥 function
                hp -= level * 5 
                print(f"戰鬥結束，剩餘 HP: {hp}")
            else:
                print(">> 安全通過，繼續深入。")
            
            current_floor += 1
            
        elif choice == "2":
            print(">> 你決定撤退回城鎮休息。")
            break
        else:
            print("無效輸入，請重新選擇。")
            
        if hp <= 0:
            print("💀 你在冒險中倒下了...")
            break
            
    if current_floor == len(dungeons):
        print("\n🎉 恭喜！你成功穿越了這個區域！")

start_adventure()

```

### 3. 劇本設定的加分技巧

即使是直線地圖，透過文字編排也能讓玩家覺得「很有戲」：

* **環境壓迫感：** 隨著 `current_floor` 增加，背景敘述可以越來越沉重。
* 第 1 階：地面平坦。
* 第 5 階：腳下開始出現白骨。


* **回城機制：** 在直線地圖中，「回城」通常意味著放棄目前的進度。你可以設計一個**「傳送石」**道具，讓玩家在特定層數存檔。
* **戰利品累積：** 玩家每前進一格，可以累積一個 `temporary_loot` 列表，如果成功通關才能領取；若中途死掉，獎勵減半或消失。這會讓「繼續前進」與「撤退」的決策變得很有壓力。

### 1. 核心邏輯：從「等待輸入」變成「偵測按鍵」

在 Pygame 的主迴圈裡，我們會偵測 `KEYDOWN` 事件。以下是一個專為「直線式迷宮」設計的架構：

```python
import pygame

# 初始化與視窗設定
pygame.init()
screen = pygame.display.set_mode((800, 600))
font = pygame.font.SysFont("simsun", 24) # 支援中文的字體

# 遊戲狀態設定
current_floor = 0
dungeons = ["平原森林", "幽暗洞窟", "灼熱火山", "最終龍宮"]
game_state = "MOVING" # 狀態：移動中、戰鬥中、對話中

running = True
while running:
    screen.fill((30, 30, 30)) # 背景深色
    
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False
        
        # 鍵盤監聽：玩家按下「向右鍵」前進
        if event.type == pygame.KEYDOWN:
            if event.key == pygame.K_RIGHT and game_state == "MOVING":
                if current_floor < len(dungeons) - 1:
                    current_floor += 1
                    print(f"前進到：{dungeons[current_floor]}")
                else:
                    print("已經到達終點！")
            
            if event.key == pygame.K_ESCAPE: # 按 ESC 離開迷宮
                print("撤退回城")
                current_floor = 0

    # 繪製畫面 (UI)
    text_surface = font.render(f"目前位置：{dungeons[current_floor]}", True, (255, 255, 255))
    screen.blit(text_surface, (50, 50))
    
    hint_surface = font.render("按 [→] 前進，按 [ESC] 撤退", True, (200, 200, 200))
    screen.blit(hint_surface, (50, 500))

    pygame.display.flip()

pygame.quit()

```

### 2. 劇本設定：如何寫得「有高級感」？

既然改用圖形化界面（即便只有文字），你的劇本寫法要配合**視覺呈現**：

#### A. 視覺化的環境描述

不要只顯示一行字，可以利用 Pygame 載入簡單的背景圖。

* **森林：** 背景稍微調綠，加入鳥叫聲。
* **龍宮：** 背景調成深藍，加入泡泡特效。
* **轉職試煉：** 背景閃爍金光。

#### B. 選單式的商店與公會

不要讓玩家打 `buy sword`。當玩家進入「武器商店」時，畫面彈出一個 List：

* 用 `pygame.Rect` 製作按鈕。
* 滑鼠游標移上去時，按鈕變色（Hover 效果）。
* 點擊直接扣錢、加裝備。
